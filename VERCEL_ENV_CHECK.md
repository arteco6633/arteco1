# КРИТИЧЕСКАЯ ПРОВЕРКА: Переменные окружения на Vercel

## Проблема
Сайт не загружается на других устройствах и мобильном интернете, только на вашем WiFi.

## Возможные причины

1. **Переменные окружения не установлены на Vercel**
2. **Переменные окружения не применены после обновления**
3. **Неправильные значения переменных окружения**

## РЕШЕНИЕ: Проверьте переменные окружения на Vercel

### Шаг 1: Откройте Vercel Dashboard
1. Зайдите на https://vercel.com
2. Войдите в аккаунт
3. Выберите проект **arteco1**

### Шаг 2: Проверьте Environment Variables
1. Перейдите в **Settings** → **Environment Variables**
2. Убедитесь, что установлены следующие переменные:

**ОБЯЗАТЕЛЬНЫЕ переменные:**
- `NEXT_PUBLIC_SUPABASE_URL` - URL вашего Supabase проекта (например: `https://xxxxx.supabase.co`)
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` - Anon key из Supabase
- `SUPABASE_SERVICE_ROLE_KEY` - Service Role key из Supabase

**ВАЖНО:** 
- Убедитесь, что значения НЕ содержат старый URL `zijajicude.beget.app`
- Убедитесь, что значения правильные (скопированы из Supabase Dashboard)

### Шаг 3: Проверьте Environment (Production, Preview, Development)
Убедитесь, что переменные установлены для **Production** (или для всех окружений):
- ✅ Production
- ✅ Preview  
- ✅ Development

### Шаг 4: После обновления переменных
1. Перейдите в **Deployments**
2. Найдите последний деплой
3. Нажмите **Redeploy** (три точки → Redeploy)
4. Дождитесь завершения деплоя

### Шаг 5: Проверьте логи деплоя
1. Откройте последний деплой
2. Перейдите на вкладку **Logs**
3. Проверьте, нет ли ошибок подключения к Supabase
4. Проверьте, что переменные окружения правильно загружены

## Как получить правильные значения из Supabase

1. Зайдите в ваш проект Supabase: https://supabase.com/dashboard
2. Выберите проект
3. Перейдите в **Settings** → **API**
4. Скопируйте:
   - **Project URL** → `NEXT_PUBLIC_SUPABASE_URL`
   - **anon public key** → `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - **service_role key** → `SUPABASE_SERVICE_ROLE_KEY` (показать ключ)

## Дополнительная диагностика

Если проблема сохраняется после обновления переменных:

1. **Проверьте консоль браузера** (F12 → Console):
   - Откройте сайт на проблемном устройстве
   - Проверьте, есть ли ошибки в консоли
   - Ищите ошибки типа "Failed to fetch" или "Network error"

2. **Проверьте Network tab** (F12 → Network):
   - Посмотрите, какие запросы не выполняются
   - Проверьте, есть ли запросы к Supabase

3. **Проверьте, что Supabase проект активен:**
   - Зайдите в Supabase Dashboard
   - Убедитесь, что проект не приостановлен
   - Проверьте, что API доступен

4. **Проверьте DNS:**
   - Убедитесь, что домен сайта правильно настроен
   - Попробуйте открыть сайт напрямую по IP (если возможно)
