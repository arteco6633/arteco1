# КРИТИЧЕСКОЕ ИСПРАВЛЕНИЕ: Загрузка сайта на мобильном интернете

## Проблема
Сайт не загружался на мобильном интернете из-за fallback на старый удаленный Supabase URL (`zijajicude.beget.app`).

## Что исправлено

1. ✅ **Убран fallback на старый URL** - теперь сайт не пытается подключиться к несуществующему серверу
2. ✅ **Убраны ссылки на старый URL** из `app/layout.tsx`
3. ✅ **Добавлена проверка переменных окружения** - теперь будет ошибка в консоли, если они не установлены

## КРИТИЧЕСКИ ВАЖНО: Проверьте переменные окружения на Vercel!

### Шаг 1: Откройте Vercel Dashboard
1. Зайдите на https://vercel.com
2. Выберите ваш проект
3. Перейдите в **Settings** → **Environment Variables**

### Шаг 2: Убедитесь, что установлены:
- `NEXT_PUBLIC_SUPABASE_URL` - URL вашего нового Supabase проекта
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` - Anon key из Supabase
- `SUPABASE_SERVICE_ROLE_KEY` - Service Role key из Supabase

### Шаг 3: После обновления переменных
1. Перейдите в **Deployments**
2. Найдите последний деплой
3. Нажмите **Redeploy** (три точки → Redeploy)

## Как проверить, что все работает

1. Откройте сайт на мобильном устройстве (не по WiFi)
2. Откройте консоль разработчика (если возможно)
3. Проверьте, нет ли ошибок подключения к Supabase
4. Убедитесь, что страница загружается

## Если проблема сохраняется

1. **Проверьте логи Vercel:**
   - Vercel Dashboard → Deployments → выберите деплой → View Function Logs

2. **Проверьте консоль браузера:**
   - Откройте DevTools (F12)
   - Вкладка Console - проверьте ошибки
   - Вкладка Network - проверьте, какие запросы не выполняются

3. **Проверьте Supabase:**
   - Убедитесь, что проект активен
   - Проверьте, что API доступен
   - Проверьте CORS настройки в Supabase Dashboard → Settings → API

## Важно

Если переменные окружения не установлены на Vercel, сайт будет показывать ошибки в консоли, но **страница все равно загрузится** (не будет блокироваться попыткой подключения к несуществующему серверу).
