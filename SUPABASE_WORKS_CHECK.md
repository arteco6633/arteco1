# Supabase проект работает!

Ошибка `{"error":"requested path is invalid"}` при открытии корневого URL Supabase - это **НОРМАЛЬНО**. 
Supabase API не предоставляет HTML-страницу по корневому URL.

Если бы проект был приостановлен, была бы другая ошибка (например, "Project not found" или "Project paused").

## Значит проблема не в статусе проекта Supabase

Ошибки 503 на сайте могут быть из-за:

1. **Неправильные ключи** - проверьте, что используете правильные ключи:
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY` = Publishable key (должен начинаться с `sb_publishable_`)
   - `SUPABASE_SERVICE_ROLE_KEY` = Secret key (должен начинаться с `sb_secret_`)

2. **Переменные окружения не применены** - после изменения на Vercel нужен Redeploy

3. **Проблема с кешем** - выполните Redeploy без кеша

4. **Проверьте логи Vercel** - могут быть ошибки компиляции или применения переменных
