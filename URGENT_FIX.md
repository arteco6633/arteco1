# СРОЧНО: Сайт работает только на вашем WiFi

## ЧТО НУЖНО СДЕЛАТЬ ПРЯМО СЕЙЧАС:

### 1. ПРИНУДИТЕЛЬНЫЙ REDEPLOY БЕЗ КЕША на Vercel

1. Зайдите на https://vercel.com
2. Ваш проект → **Deployments**
3. Найдите последний деплой
4. Нажмите **три точки (⋮)** → **Redeploy**
5. ВАЖНО: Снимите галочку **"Use existing Build Cache"** (если есть)
6. Нажмите **Redeploy**
7. Дождитесь завершения (обычно 2-3 минуты)

### 2. ОЧИСТИТЕ КЕШ БРАУЗЕРА на проблемном устройстве

**Android:**
- Chrome: Настройки → Конфиденциальность → Очистить данные просмотра → Кеш → Очистить
- ИЛИ используйте режим инкогнито

**iPhone/iPad:**
- Safari: Настройки → Safari → Очистить историю и данные
- ИЛИ используйте режим приватного просмотра

### 3. ПРОВЕРЬТЕ на проблемном устройстве

1. Откройте сайт в режиме инкогнито/приватного просмотра
2. Если возможно, откройте DevTools (F12)
3. Перейдите на вкладку **Network**
4. Обновите страницу
5. Посмотрите, куда идут запросы:
   - ✅ Должны идти к вашему новому Supabase (например, `xxxxx.supabase.co`)
   - ❌ НЕ должны идти к `zijajicude.beget.app`

### 4. ЕСЛИ НЕ ПОМОГАЕТ - проверьте переменные еще раз

1. Vercel Dashboard → Settings → Environment Variables
2. Убедитесь, что `NEXT_PUBLIC_SUPABASE_URL` НЕ содержит `zijajicude.beget.app`
3. Если содержит - замените на правильный URL
4. Выполните Redeploy (шаг 1)

## Почему работает только на вашем WiFi?

- Ваш браузер кеширует старую версию сайта
- Ваш WiFi использует кешированный DNS
- Vercel CDN кеширует старую версию для других локаций

**Решение:** Принудительный redeploy без кеша + очистка кеша браузера
