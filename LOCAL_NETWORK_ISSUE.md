# Проблема: Сайт работает только на вашем WiFi (одна сеть)

## Если все устройства в одном регионе, то проблема НЕ в региональном кеше

## Возможные причины:

### 1. Кеш браузера на вашем устройстве (НАИБОЛЕЕ ВЕРОЯТНО)
Ваш браузер кеширует старую рабочую версию сайта с правильными данными.

**Проверка:**
- Откройте сайт на вашем устройстве в режиме инкогнито/приватного просмотра
- Если в инкогнито НЕ работает - значит проблема в кеше браузера

**Решение:**
- Очистите кеш браузера на проблемных устройствах
- ИЛИ используйте режим инкогнито для проверки

### 2. DNS кеш на вашем роутере/WiFi
Ваш WiFi роутер может кешировать DNS записи.

**Проверка:**
- На проблемном устройстве откройте сайт
- Откройте DevTools → Network
- Посмотрите, к какому IP адресу идут запросы
- Сравните с вашим устройством

**Решение:**
- Перезагрузите WiFi роутер
- ИЛИ на проблемных устройствах используйте другой DNS (8.8.8.8)

### 3. Проблема с Supabase доступом по IP
Supabase может блокировать запросы с определенных IP адресов или устройств.

**Проверка:**
- На проблемном устройстве откройте DevTools → Network
- Посмотрите на запросы к Supabase (должны быть к `iwamliovlphpfnghqngn.supabase.co`)
- Проверьте статус ответов - если 503, значит Supabase блокирует

**Решение:**
- Проверьте настройки Supabase → Settings → API
- Убедитесь, что нет IP-ограничений
- Проверьте CORS настройки

### 4. Проблема с User-Agent или заголовками
Некоторые устройства могут отправлять заголовки, которые блокируются.

**Проверка:**
- На проблемном устройстве откройте DevTools → Network
- Откройте любой запрос к Supabase
- Посмотрите на заголовки запроса (Request Headers)
- Сравните с вашим устройством

### 5. Проблема с SSL/сертификатом
Могут быть проблемы с SSL-сертификатом для некоторых устройств/браузеров.

**Проверка:**
- На проблемном устройстве откройте DevTools → Console
- Проверьте, нет ли ошибок SSL/сертификата

## Диагностика:

На проблемном устройстве:
1. Откройте сайт в режиме инкогнито
2. Откройте DevTools (F12) → Network
3. Обновите страницу
4. Найдите запросы к Supabase (должны быть к `iwamliovlphpfnghqngn.supabase.co`)
5. Откройте любой запрос и посмотрите:
   - Статус ответа (503? 200?)
   - URL запроса (правильный?)
   - Заголовки запроса
   - Ответ сервера (Response)

Сообщите, что вы видите в Network tab на проблемном устройстве.
